# 大语言模型(LLM)



## **LLM 四大核心能力**

1. 涌现能力 (Emergent Abilities) —— **“量变引起质变”**
   - **核心：** 当模型参数规模达到一定量级（通常是百亿级以上）时，会突然展现出小模型所不具备的能力。
   - **类比：** 就像水温从 99°C 升到 100°C 突然沸腾（相变）。
   - **意义：** 这是学术界对 LLM 充满信心的根源，相信随着规模增大，它能逼近通用人工智能（AGI）。

2. 上下文学习 (In-context Learning) —— **“现学现卖”**
   - **核心：** 只要在提问时给它几个例子（Few-shot）或一段说明，它就能学会处理新任务，**不需要**重新训练或修改参数。
   - **优势：**
     - **省钱：** 告别了传统 PLM 需要高昂算力进行“微调”的时代。
     - **省事：** 研发范式从“写代码练模型”转向了 **Prompt Engineering**（提示词工程）。

3. 指令遵循 (Instruction Following) —— **“听得懂人话”**
   - **核心：** 经过“指令微调”后，模型能理解并执行从未见过的指令，具备极强的**泛化能力**。
   - **应用：** 比如 ChatGPT。你不需要教它怎么写作文或改代码，直接下令即可。
   - **地位：** 它是目前 AI Agent（智能体）和全能助理的技术底座。
4. 逐步推理 (Step-by-Step Reasoning) —— **“逻辑思考”**
   - **核心：** 通过 **思维链（CoT, Chain-of-Thought）** 技术，模型可以像人类一样一环扣一环地推导复杂问题。
   - **突破：** 解决了以前 AI 无法处理复杂数学题、逻辑陷阱的难题。
   - **猜测：** 业界普遍认为这种逻辑能力可能源于大规模代码数据的训练。



## **LLM特性**

1. 多语言支持 (Multilingualism) —— **“天生的翻译官”**
   - **现状：** 由于预训练数据包含全球网页，LLM 天然具备跨语言能力。
   - **差异：** 英文语料质量最高，所以大部分模型英文最强。国内模型（如通义千问）在中文语境、成语、文化背景上通常做了专门优化，表现更地道。

2. 长文本处理 (Long Context) —— **“量子波动速读”**
   - **核心技术：** 传统模型只能读几百字（512 tokens），而 LLM 通过 **RoPE（旋转位置编码）** 等技术，可以处理几万甚至几十万字。
   - **外推能力：** 即使模型训练时只看过 3.2 万字，推理时也能通过算法技巧处理更长的文本（如 200k）。
   - **应用：** 这让 AI 读完一整本书并写总结、分析超长合同成为可能。

3. 拓展多模态 (Multimodality) —— **“打破次元壁”**
   - **原理：** 通过 **Adapter（适配器）** 或图像编码器，将图片、视频信息“翻译”成 LLM 能理解的信号。
   - **进化：** LLM 不再只宅在文字世界，它开始能“看”图说话、理解视频，甚至未来能听懂声音。

4. 挥之不去的“幻觉” (Hallucination) —— **“一本正经地胡说八道”**
   - **定义：** 模型生成看似专业、逻辑通顺但事实错误的信息（如虚构参考文献）。
   - **挑战：** 它是 LLM 的固有缺陷，在医疗、金融等严谨领域非常危险。
   - **对策：** 目前主要靠 **RAG（检索增强生成）**，即让模型先查资料再回答，来减轻这个问题。



## 如何训练一个 LLM

一般而言，训练一个完整的 LLM 需要经过图1中的三个阶段——Pretrain、SFT 和 RLHF



### Pretrain

LLM 生产流程中最“重型”的阶段——**预训练（Pretrain）**

类似于“义务教育阶段“



1. 核心任务：因果语言模型 (CLM)
   - **原理：** 就像“完形填空”，给定上文，预测下一个字（Next Token Prediction）。
   - **架构：** 几乎统一采用 **Decoder-Only**（如 LLaMA、GPT 系列）。

2. LLM vs 传统 PLM (以 BERT 为例)

   LLM 并不是简单的“放大版”，其规模和资源消耗是指数级增长的：

   - **参数量：** BERT 只有 1亿-3亿，而 LLM 动辄 **百亿（10B）到千亿（175B）**。
   - **数据量：** 满足 **Scaling Law（尺度定律）**，训练数据量通常是参数量的 1.7 到 20 倍（达到 T 级别，即万亿 token）。
   - **算力：** 训练百亿级模型通常需要上千张 A100 显卡连续工作一个月。

3. 分布式训练：解决“存不下”和“算得慢”

   由于单张显卡塞不下巨大的模型，必须使用分布式框架（如 **DeepSpeed**）：

   - **数据并行：** 多个显卡各练各的数据，最后汇总梯度。
   - **模型并行：** 把模型“劈开”，不同显卡负责不同的层或参数。
   - **ZeRO 优化器（核心黑科技）：**  **ZeRO-1/2/3：** 逐步将优化器状态、梯度、参数分片存储到不同显卡上，极大地节省了显存占用。

4. 数据：LLM 的“营养来源”
   - **配比是核心：** 像配方一样，混合网页（CommonCrawl）、代码（Github）、百科（Wikipedia）和论文（ArXiv）。
   - **质量大于数量：** 干净的 600B 数据效果往往优于杂乱的 1T 数据。
   - **处理流程：** 
     1. **文档准备：** 提取纯文本，过滤有害 URL。
     2. **语料过滤：** 删掉广告、乱码。
     3. **语料去重：** 删掉高度相似的内容，防止模型“学死板”（影响泛化）。

目前，已有很多经过处理的高质量预训练语料和专用于预训练数据处理的框架

例如，有基于 LLaMA 思路收集、清洗的预训练数据集[RedPajama-1T](https://huggingface.co/datasets/togethercomputer/RedPajama-Data-1T)

以及在 RedPajama 基础上进行筛选去重的[SlimPajama-627B](https://huggingface.co/datasets/cerebras/SlimPajama-627B/tree/main/train)数据集

实验证明高质量的 627B Slimpajama 数据集能够获得比 1T 的 RedPajama 数据集更好的效果



### SFT

 LLM 训练的第二个关键阶段：**SFT（有监督微调）**

通俗理解：SFT 就是在教它“**如何听懂人话并按规矩办事**”



**SFT 的核心目标：从“背书”到“对话”**

- **本质区别：** 预训练模型（Base 模型）只会无休止地接龙下一个词；SFT 模型（Chat/Instruct 模型）则学会了**指令遵循**，即理解用户的意图并给出对应回复。
- **范式转移：** 传统微调是“一个任务一个模型”（如专门练一个分类模型）；LLM 的 SFT 是“**通用指令微调**”，旨在训练一种泛化的处理能力



不同于预训练使用的无监督语料，SFT 使用的指令数据集是有监督语料

除去设计广泛、合理的指令外，还需要对指令回复进行人工标注，并保证标注的高质量



**构建高质量的数据集是SFT的核心**

A. 数据结构（JSON 示例）

每一条 SFT 数据通常被格式化为“指令（Instruction）+ 输入（Input）+ 输出（Output）”：

```
{
    "instruction": "请分析这段文字的情感倾向",
    "input": "今天考试没考好，心情很郁闷。",
    "output": "这段文字表达了消极、低落的情感。"
}
```

B. 模板拼接（Prompt Template）

为了让模型区分指令和回答，训练时会加上**特殊占位符**。

不同模型模板不同（如 LLaMA-3, Qwen, Alpaca）最常见的是： 

`### Instruction:\n{{content}}\n\n### Response:\n`



注意，因为指令微调本质上仍然是对模型进行 CLM 训练

只不过要求模型对指令进行理解和回复而不是简单地预测下一个 token

所以模型预测的结果不仅是 output，而应该是 input + output

只不过 input 部分不参与 loss 的计算，但回复指令本身还是以预测下一个 token 的形式来实现的



**多轮对话的训练精髓：全量预测 (Full Prediction)**

这是你作为开发者最需要理解的技巧。模型如何记住聊天历史？

- **构造逻辑：** 将 $Q_1A_1 + Q_2A_2 + Q_3A_3$ 拼成一长条文本。
- **计算掩码 (Loss Mask)：** 在训练时，虽然整段话都喂给了模型，但我们只对 $A_1, A_2, A_3$ 计算损失（Loss）。
  - 对于所有的用户问题 $Q_1, Q_2, Q_3$，模型预测它们产生的 Loss 会被 **Mask（屏蔽）** 掉。
- **原因：** 我们希望模型学习“怎么回答”，而不是学习“用户会问什么”



### RLHF

RLHF，全称是 Reinforcement Learning from Human Feedback，即人类反馈强化学习

类比：**RLHF** 就是在为模型建立“**三观**”



1. RLHF 的核心使命：对齐（Alignment）
   - **定义的转变：** 模型不仅要“会答题”（SFT），还要答得“像人一样好”。
   - **三大标准：** 追求 **HHH** 标准，即 **Helpful**（有用）、**Honest**（诚实）、**Harmless**（无害）。
   - **为什么要强化学习？** 传统的 SFT（监督学习）只能让模型学会“预测标准答案”，但人类对好坏的判断是主观且复杂的（比如：两份答案都对，但哪份更礼貌？）。强化学习能让模型在没有“唯一正确答案”的情况下，通过不断尝试和反馈，探索出人类最喜欢的表达方式。

------

2. 第一步：训练奖励模型 (RM, Reward Model) —— “数字化的人类裁判”

   在让模型大规模练习之前，需要先训练一个“裁判”来代替人类打分（因为人类不可能实时给百万次生成打分）。

   - **本质：** RM 是一个文本分类模型，它输入一段对话，输出一个代表“好坏程度”的分数（标量）。
   - **训练数据的特殊性：**
     - **排序数据：** 不让标注员给回复打分（因为 80 分和 85 分很难界定），而是让他们做“选择题”。
     - **格式：** 给出一个 Prompt，展示两个回复（Chosen 和 Rejected）。
     - **逻辑：** 训练 RM，使其在遇到 `Chosen` 回复时给高分，遇到 `Rejected` 回复时给低分。
   - **难点：** 标注员之间的主观差异。通过大规模的排序数据，RM 能够学到人类社会的“公约数”偏好。

------

3. 第二步：PPO 训练 —— “带着镣铐的自我进化”

   这是真正的强化学习阶段。为了保证模型在变强的同时“不跑偏”，PPO 过程中动用了四个模型：

   - **四大模型的分工：**
     1. **Actor Model（演员）：** 正在学习的模型，负责生成回复（我们要更新的对象）。
     2. **Ref Model（参考）：** 冻结的 SFT 模型，负责盯着演员，别让它为了拿高分而变得语无伦次。
     3. **Reward Model（奖励）：** 冻结的裁判，给演员的表现打分。
     4. **Critic Model（评论家）：** 辅助预测每个阶段能拿多少分，稳定训练过程。
   - **关键机制：KL 散度限制。**
     - 如果 Actor 生成的内容和 Ref 偏离太远，系统会给一个惩罚（Penalty）。这防止了模型为了“讨好”奖励模型而出现崩溃（比如通过作弊手段刷分）。

------

4. RLHF 的工程挑战：高门槛与高消耗
   - **显存黑洞：** 同时加载四个大模型，显存需求是 SFT 的 4 倍。即使是 7B 模型，也需要数张 80G A100 才能跑起来。
   - **训练极度不稳定：** 强化学习对超参数（如学习率、KL 权重）非常敏感，稍有不慎模型就会“训飞”或变成只会重复一句话的复读机。

------

5. 新趋势：DPO（直接偏好优化） —— “化繁为简”

   由于 RLHF 太难、太贵，DPO 成了目前的行业新宠：

   - **原理：** 通过数学推导，证明了不需要训练专门的奖励模型，也不需要复杂的 PPO 算法，直接通过 **监督学习（SFT）的思路** 就能完成偏好对齐。
   - **优势：** 只需 2 个模型（Actor 和 Ref），简单稳定，显存占用低，是目前大多数开源模型（如 Llama-3、Qwen）首选的对齐方案。
