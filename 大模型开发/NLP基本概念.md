# NLP基本概念



> 概念：NLP 是 一种让计算机理解、解释和生成人类语言的技术。它是人工智能领域中一个极为活跃和重要的研究方向，其核心任务是通过计算机程序来模拟人类对语言的认知和使用过程



## NLP任务



### 中文分词

NLP 领域中的一个**基础任务**

目的是将连续的中文文本切分成有意义的词汇序列

```
英文输入：The cat sits on the mat.
英文切割输出：[The | cat | sits | on | the | mat]
中文输入：今天天气真好，适合出去游玩.
中文切割输出：["今天", "天气", "真", "好", "，", "适合", "出去", "游玩", "。"]
```

正确的分词结对于后续的词性标注、实体识别、句法分析等任务至关重要



### 字词切分

NLP 领域中的一种常见的**文本预处理技术**

旨在将词汇进一步分解为更小的单位

特别适用于处理词汇稀疏问题

当遇到罕见词或未见过的新词时，能够通过已知的子词单位来理解或生成这些词汇

在处理那些拼写复杂，预训练语言模型尤为重要

```
输入：unhappiness

不使用子词切分：整个单词作为一个单位，输出：“unhappiness”
使用子词切分（假设BPE算法）：单词被分割为：“un”、“happi”、“ness”
```



**字词切分的方法**

字词切分的目标是将连续的文本流切分为具有语义的子串（Token）。根据切分粒度的不同，主要分为三类： 

- **词粒度（Word-level）：**
  - **原理：** 以完整的词作为切分单位。
  - **中文分词：** 由于中文缺乏天然空格，常用方法包括基于词典的匹配（如正向/逆向最大匹配）、基于统计的模型（如 HMM、CRF）以及深度学习方法（如 BiLSTM）。
  - **缺点：** 词表过大，容易出现“词汇外”（OOV）问题，无法处理新词。
- **字/字符粒度（Character-level）：**
  - **原理：** 将文本拆分为单个汉字或英文字母。
  - **优缺点：** 词表极小（如中文约几千字），无 OOV 问题，但单个字符往往缺乏完整的语义信息。
- **子词粒度（Subword-level）：**
  - **原理：** 介于字与词之间。常用词不拆分，罕见词拆分为有意义的子块。
  - **常用算法：**
    - **BPE (Byte Pair Encoding):** 通过合并高频字符对来构建词表，广泛用于 GPT 系列。
    - **WordPiece:** 类似 BPE，但基于概率最大化进行合并，常用于 BERT。
    - **SentencePiece:** 将空格也视为字符，实现语言无关的切分，适用于 T5、XLNet。 



**预训练语言模型 (PLM)**

**预训练语言模型**是一种先在海量无标注数据（如互联网文本、书籍）上进行大规模训练，学习通用语言规律，然后再通过“微调”应用于特定任务的机器学习模型。 

- **核心理念：** “先学习常识，再学习专业技能”。通过大规模预训练，模型能够学习到通用的特征表示（Embedding），显著提高在下游任务（如翻译、问答、分类）中的准确性和泛化能力。
- **训练方式：**
  - **自监督学习：** 利用文本自身作为标签。例如，随机遮盖一部分词让模型预测（掩码语言模型，如 [BERT](https://zhuanlan.zhihu.com/p/607605399)），或者根据上文预测下一个词（因果语言模型，如 GPT）。
- **典型模型：**
  - **理解型：** BERT（双向编码，擅长文本分类、问答）。
  - **生成型：** GPT 系列（单向解码，擅长文本创作、对话）。
  - **通用型：** T5（将所有任务转化为文本到文本的格式）。 



### 分词算法

**BPE**

一种广泛应用于自然语言处理（NLP）中的子词（Subword）分词算法

1. 核心原理

BPE 的核心思想是：**从字符级开始，通过迭代不断合并语料库中出现频率最高的相邻字符对（或子词对），逐步构建词表**。 

2. 算法步骤

BPE 的构建过程通常包括以下步骤：

1. **初始化**：将语料库中的单词拆分为字符序列，并统计各单词出现的频次。
2. **统计频率**：统计所有相邻单位（如字符对）在语料中出现的总次数。
3. **合并最高频对**：找到出现频率最高的相邻对，将其合并为一个新的子词，并更新词表。
4. **循环迭代**：重复上述过程，直到达到预设的词表大小（Vocabulary Size）或达到最大迭代次数。 

**示例说明：**
假设语料中有 `hug: 10次` 和 `pug: 5次`。

- 初始序列为 `h u g` 和 `p u g`。
- 相邻对 `u g` 出现了 15 次（最高），合并为 `ug`。
- 此时单词变为 `h ug` 和 `p ug`，词表新增 `ug`。

3. 主要优点 

- **解决 OOV（词汇外）问题**：即使遇到从未见过的单词，也可以将其拆分为已知的子词（如把 "unhelpful" 拆分为 "un", "help", "ful"），模型依然能处理。
- **压缩词表规模**：相比纯词级别分词，BPE 能用更小的词表覆盖更广的语义空间。
- **形态语义捕捉**：能够自动学习到一些常见的前缀、后缀和词根。

4. 局限性 

- **基于统计**：对训练语料的依赖性强，如果语料规模过小，生成的子词可能缺乏代表性。
- **贪心性质**：分词过程是贪心的，有时无法保证生成全局最优的切分序列。



**WordPiece**

WordPiece 与 BPE 的训练流程极其相似，唯一的、也是最关键的区别在于**合并准则**

1. 核心原理：互信息（Likelihood）

   BPE 只看“谁出现的次数多”，而 WordPiece 看“这两个词拼在一起，是否比它们分开出现更合理”。
   它计算合并后的**互信息**：P(AB)/P(A)P(B)

- 如果**A和B**总是形影不离（如 `ice` 和 `cream`），分母小，得分高，合并。

- 如果**A或B**经常单独乱跑（如 `a` 这个字母)，分母变大，得分低，不急着合并。

  

2. 算法步骤
   1. **初始化**：将词表设为所有字符。
   2. **训练语言模型**：在当前词表下，建立一个简单的语言模型。
   3. **贪心合并**：遍历所有可能的组合，选择合并后能最大程度增加**训练数据似然概率**的那一对。
   4. **循环**：直到词表达到预设规模。



3. 标识特征

   WordPiece 在处理非词首子词时会加上特殊标记（如 `##`）。

- **示例**：`unhappiness` ![img](data:image/gif;base64,R0lGODlhAQABAIAAAP///wAAACH5BAEAAAAALAAAAAABAAEAAAICRAEAOw==) `un`, `##happi`, `##ness`
- **优点**：通过这种方式，模型能分清 `un` 是一个独立的词，还是一个词的后缀。



**Unigram**

Unigram 走的是一条完全相反的道路：**“先全都要，再慢慢扔”**

1. 核心原理：概率评估

   它假设文本中的每个子词都是独立出现的。它不通过合并来构建，而是从一个巨大的候选池（比如所有的子串）开始，逐步剔除掉对语料库解释力最弱的词。

2. 算法步骤
   1. **海量初始化**：利用所有的单词和足够长的高频子串构建一个巨大的初始词表。
   2. **EM 算法优化**：
      - **E-step**：固定当前词表，计算每种切分方案的概率。
      - **M-step**：计算如果去掉某个子词，整个语料库的损失（Loss）会增加多少
   3. **瘦身**：剔除损失增加最少（即“没它也行”）的那部分子词（通常一次删 10%-20%）。
   4. **循环**：直到词表大小符合要求。

3. 核心优势：子词采样（Subword Regularization）

   这是 Unigram 最强的地方。因为它有概率分布，训练时可以**随机**选择不同的切分方式（比如 "playing" 有时切成 `play/ing`，有时切成 `pla/ying`）。这种**随机扰动**能让模型学到的语义表达极其稳健。

   

**SentencePiece**

SentencePiece 严格来说不是新算法，而是一个**集成框架**，它是 Google Open Source 的杰作。

1. 为什么它是现在的“标配”？

   在 BPE 或 WordPiece 的早期实现中，必须先进行“预分词”（用空格把词分开）。但这会带来两个麻烦：

   - **中文噩梦**：中文没空格，得先求助 jieba 等工具，分词准不准会直接拖累后面的模型。
   - **无法还原**：由于空格在分词时被丢弃了，`New York` 分完词再拼回去可能变成 `NewYork`

2. 核心创新：空格即字符

   SentencePiece 巧妙地把**空格当作一个普通字符**（通常用 `_` 表示）。

   - **输入**：`Hello world`
   - **内部处理**：`_Hello_world`
   - **分词后**：`[_Hello, _world]`
   - **无损还原**：只要把 Token 连起来，再把 `_` 换回空格。这意味着它可以直接处理 **Raw Text（原始文本）**，不需要任何复杂的预分词规则

3. 实现灵活性

   你可以指定它内部运行 **BPE** 模式（如 Llama 3 方案）或者 **Unigram** 模式（如 T5 方案）



### 词性标注

NLP 领域中的一项**基础任务**

为文本中的每个单词分配一个词性标签

为文本中的每个单词分配一个词性标签



**词性标注的技术演进主要经历了以下阶段**：

1. **基于规则的方法**：利用人工编写的语法规则和词典进行消歧。
2. **基于统计的方法**：如隐马尔可夫模型 (HMM)和条件随机场 (CRF)，通过语料库计算概率来预测标注序列。
3. **基于深度学习的方法**：目前的主流技术，常用模型包括 [BiLSTM+CRF](https://github.com/NLP-LOVE/Introduction-NLP/blob/master/chapter/7.词性标注.md) 或基于 Transformer 架构的模型（如 BERT），能够更有效地捕获长距离的上下文信息



### 文本分类

NLP 领域的一项**核心任务**

旨在自动识别文本中具有特定意义的实体

它就像是给海量信息建立一套“自动分拣系统”，通过分析文本的语义内容来分配最合适的标签

```
输入：李雷和韩梅梅是北京市海淀区的居民，他们计划在2024年4月7日去上海旅行。

输出：[("李雷", "人名"), ("韩梅梅", "人名"), ("北京市海淀区", "地名"), ("2024年4月7日", "日期"), ("上海", "地名")]
```



**技术演进与常见算法**

文本分类技术主要分为两个发展阶段： 

1. 传统机器学习（浅层学习）

   这些方法通常依赖于手工特征提取（如 TF-IDF）： 

   - **朴素贝叶斯 (Naive Bayes)**：基于贝叶斯定理，计算简单且在小规模文本上表现良好。
   - **支持向量机 (SVM)**：擅长处理高维特征空间，曾是文本分类的基准模型。
   - **随机森林 (Random Forest)**：通过集成多个决策树来提高分类的准确性和稳定性。

2. 深度学习（主流技术）

   利用神经网络自动捕捉文本的深层特征，无需繁杂的人工特征工程： 

   - **RNN/LSTM**：擅长处理序列数据，能够捕获文本的上下文信息。
   - **CNN (TextCNN)**：利用卷积核提取局部特征，在处理短文本分类时速度极快且效果显著。
   - **Transformer (BERT/RoBERTa)**：目前业内表现最强的模型，通过预训练大规模语料，对语义的理解达到了极高的精度。 

**基本工作流程**

一个标准的文本分类项目通常包含以下步骤： 

1. **数据采集**：收集并清洗带有类别标签的语料。
2. **文本预处理**：包括分词、去停用词、清洗特殊字符等。
3. **特征表示**：将文本转化为计算机能读懂的向量（如 Word2Vec 或 Embedding 层）。
4. **模型训练**：使用标注数据训练分类器。
5. **评估与部署**：通过准确率、召回率等指标评估模型性能，并应用到实际业务

### 文本摘要

NLP 中的一个**重要任务**



根据生成方式的不同，文本摘要可以分为两大类：**抽取式摘要**和**生成式摘要**

- 抽取式摘要：抽取式摘要通过直接从原文中选取关键句子或短语来组成摘要。优点是摘要中的信息完全来自原文，因此准确性较高。然而，由于仅仅是原文中句子的拼接，有时候生成的摘要可能不够流畅
  - **原理**：像用“荧光笔”划重点。从原文中直接挑选最重要的句子或短语，重新组合成摘要。
  - **优点**：生成速度快，语法通顺（因为句子是原生的），不会胡编乱造。
  - **常用算法**：TextRank（类似网页排名的逻辑）、基于聚类的方法。
- 生成式摘要：与抽取式摘要不同，生成式摘要不仅涉及选择文本片段，还需要对这些片段进行重新组织和改写，并生成新的内容。生成式摘要更具挑战性，因为它需要理解文本的深层含义，并能够以新的方式表达相同的信息。生成式摘要通常需要更复杂的模型，如基于注意力机制的序列到序列模型（Seq2Seq）
  - **原理**：像“读后感”。机器先理解全文语义，然后用自己的语言重新组织表达，可能会出现原文中没有的新词。
  - **优点**：更接近人类的行为，压缩比高，表达更自然灵活。
  - **常用算法**：基于 Transformer 架构的预训练模型，如 BART、T5 以及现在的 **GPT 系列**大模型



**评价指标**

评价摘要好坏不能只靠感觉，NLP 领域常用 **ROUGE** 指标（Recall-Oriented Understudy for Gisting Evaluation），通过计算生成摘要与人工参考摘要之间的字词重叠度（N-gram）来打分

**常见挑战**

- **幻觉问题 (Hallucination)**：生成式摘要可能会凭空捏造事实（说瞎话）。
- **指代模糊**：摘要中出现“他”、“该机构”时，读者可能不清楚具体指代谁。
- **长文本处理**：面对长达万字的文档，如何不丢失细节且保持逻辑连贯。



### 自动问答

NLP 领域中的一个**高级任务**

使计算机能够理解自然语言提出的问题，并根据给定的数据源自动提供准确的答案

自动问答大致可分为三类：

1. 检索式问答：检索式问答通过搜索引擎等方式从大量文本中检索答案
2. 知识库问答：知识库问答通过结构化的知识库来回答问题
3. 社区问答：社区问答则依赖于用户生成的问答数据，如问答社区、论坛等



### 语言模型

NLP 领域中的一个**核心技术**

本质上是一个用于预测文本序列中下一个词或字符出现概率的数学模型



- **统计语言模型（N-gram）**：基于马尔可夫假设，通过统计词组出现的频率来预测后续词。
- **神经语言模型（RNN/LSTM）**：引入神经网络处理长距离依赖，但计算效率较低。
- **大语言模型（LLM）**：基于 [Transformer 架构](https://developers.google.com/machine-learning/crash-course/llm/transformers?hl=zh-cn)，通过自注意力机制（Self-Attention）处理海量参数和数据，实现了理解力的质变





## 文本表示

将文本数据数字化，使计算机能够对文本进行有效的分析和处理

NLP 领域中的一项基础性和必要性工作，它直接影响甚至决定着 NLP 系统的质量和性能



### 词向量

**词嵌入（Word Embedding）与词向量（Word Vector）**

简单来说，词嵌入是一门技术/过程，而词向量是它的产物



**向量空间模型（VSM）**是 NLP 领域中一个基础且强大的文本表示方法

在模型中，每个维度代表一个特征项，而向量中的每个元素值代表该特征项在文本中的权重



**缺陷**

- **维度爆炸与稀疏**：词汇表有几万个词，向量就有几万维，但一句话只有几个词。计算机在处理这种“全是零”的矩阵时，效率极低，且浪费空间。
- **语义孤岛**：在 VSM 里，“荷花”和“莲花”是两个完全独立的维度。模型无法识别它们其实是一个东西，因为每个词都被视为**互相独立**的。
- **“乱炖”逻辑**：它只管词出没出现，不管顺序。换句话说，“我不爱你”和“爱我不你”在 VSM 看来是完全一样的

**研究方向**

- **降维与提炼**：通过 **SVD（奇异值分解）** 或 **LDA（主题模型）**，把几万维的稀疏向量压缩成几十维的“精华向量”。这就是从 VSM 向 **LSA（潜在语义分析）** 的跨越。
- **权重优化**：不再只是数次数。比如利用图算法看哪个词是“核心节点”，或者结合业务场景调整权重，让坐标值更能反映真实语义



### 词嵌入

常见的词嵌入方法演变

1. **静态词嵌入**：如 Word2Vec 和 GloVe。每个词对应一个固定向量。
2. **动态词嵌入**：如 BERT 和 ELMo。同一个词在不同语境下（如“苹果手机”和“吃个苹果”）会生成不同的向量。



#### Word2Vec

基于神经网络NNLM的语言模型，旨在通过学习词与词之间的上下文关系来生成词的密集向量表示

Word2Vec的核心思想是利用词在文本中的上下文信息来捕捉词之间的语义关系，从而使得语义相似或相关的词在向量空间中距离较近

**两种训练架构**

Word2Vec 主要包含两种互为镜像的模型架构： 

- **CBOW (Continuous Bag-of-Words)**：**用上下文预测中心词**。它将周围词的向量加和或取平均，以此预测中间缺失的词。这种模型在训练速度上较快，适合小型语料库。
- **Skip-gram**：**用中心词预测上下文**。它输入一个词，尝试准确预测其前后一定范围内的背景词。该架构在大型语料中表现更好，对低频词的处理效果通常优于 CBOW



#### GloVe

**静态词向量**的代表作

**实现原理**：它基于“全局矩阵分解”和“局部上下文窗口”。简单来说，它会统计整个语料库中单词共同出现的频率（共现矩阵），然后通过损失函数让向量的距离能够反映词与词之间的语义关系。

**核心特点**：

- **静态性**：训练完后，每个词对应一个**固定**的向量。无论“苹果”是指水果还是手机，它的向量永远不变。
- **效率高**：计算速度快，向量占用空间小，适合简单的分类任务。
- **语义关系**：能很好地捕捉词语间的类比关系，比如`v(king)−v(man)+v(woman)≈v(queen)`



#### ELMo

一种的NLP模型

有效解决了传统静态词向量（如 Word2Vec 和 GloVe）无法处理**一词多义**的问题



**ELMo 核心原理**

ELMo 的核心思想是，一个词的含义会根据其在句子中的具体语境而变化。它使用**深度双向长短期记忆网络（bi-LSTM）模型**来实现这一点

**ELMo采用典型的两阶段过程: **

- 第1个阶段是利用语言模型进行预训练;
- 第2个阶段是在做特定任务时, 从预训练网络中提取对应单词的词向量作为新特征补充到下游任务中。基于RNN的LSTM模型训练时间长, 特征提取是ELMo模型优化和提升的关键

**核心原理**

- **双向语言模型（biLM）**：ELMo 的预训练过程涉及一个双向语言模型，该模型同时考虑单词的左侧（前向 LSTM）和右侧（后向 LSTM）上下文。这使得模型能够捕捉到完整的句子信息，而不仅仅是单向的序列。
- **深层语境**：与仅使用最后一层隐藏状态不同，ELMo 将 bi-LSTM **所有隐藏层**的表示进行加权组合。这允许下游任务根据需要提取不同层次的语言信息（例如，底层可能捕捉句法特征，高层可能捕捉语义特征）。
- **字符级输入**：ELMo 使用字符卷积网络作为输入层，这意味着它可以处理训练集中未出现过的词（OOV 问题），使其具有更强的泛化能力



#### BERT 

**动态语境化**词表示的集大成者

- **实现原理**：
  - **Transformer 架构**：BERT 完全抛弃了 LSTM，使用 **Self-Attention（自注意力机制）**。这让它能同时关注到句子里所有单词，处理长距离依赖的能力极强。
  - **双向编码**：不同于传统模型从左到右或从右到左，BERT 通过“掩码语言模型”（MLM，即完形填空）同时学习左右两侧的语境。
- **核心特点**：
  - **动态性**：同一个词在不同句子里，其向量是根据上下文实时生成的。
  - **预训练 + 微调**：先在海量数据上学习通用的语言规律，然后针对特定任务（如翻译、搜索、问答）进行微调。
