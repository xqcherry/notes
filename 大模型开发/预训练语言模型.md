# 预训练语言模型（PLM）

> **预训练语言模型 = 通用的语言大脑 + 具体的任务插件**

1. 预训练 (Pre-training)：打底子

   模型在**海量**的互联网文本（如维基百科、书籍、代码）上进行“自学”

   - **学什么**：学习语言的语法、逻辑、常识和词汇间的关系
   - **怎么学**：通常玩“完形填空”或“猜下一个词”的游戏。比如输入“床前明月光，疑是地上[ ]”，模型要学会填出“霜”
   - **结果**：学完后，模型已经拥有了极强的**通用语言能力**，但还不擅长具体的任务（如翻译或判案）

2. 微调 (Fine-tuning)：练专项

   在特定的任务数据上对模型进行小规模训练

   - **例子**：如果你想做一个“医疗咨询机器人”，就拿医学论文和病例给这个“博学”的模型再学一遍
   - **优势**：因为模型已经懂语言了，所以只需要少量专业数据就能快速上手



**基于 Transformer 的不同组件（Encoder/Decoder）**

衍生出了三条各具优势的技术路线

1. **Encoder-Only（代表：BERT）**
   - **特点**：利用 Encoder 的双向上下文理解能力。
   - **任务**：擅长**自然语言理解 (NLU)**，如分类、标注、阅读理解。
   - **创新**：引入了 **掩码语言模型 (MLM)**，让模型能“瞻前顾后”。
2. **Decoder-Only（代表：GPT 系列）**
   - **特点**：利用 Decoder 的单向从左往右生成的特性。
   - **任务**：专注于**自然语言生成 (NLG)**，是当前大模型 (LLM) 的绝对主流。
   - **策略**：通过海量参数和语料，走**通用人工智能 (AGI)** 的路径。
3. **Encoder-Decoder（代表：T5）**
   - **特点**：保留完整结构，将所有任务（翻译、摘要等）统一转化为 **Text-to-Text** 格式。
   - **优势**：在需要输入输出高度转换的任务中表现极佳且灵活。
4. **典型模型：**
   - **理解型：** BERT（双向编码，擅长文本分类、问答）。
   - **生成型：** GPT 系列（单向解码，擅长文本创作、对话）。
   - **通用型：** T5（将所有任务转化为文本到文本的格式）。



## BERT

**一种自然语言处理（NLP）模型，它通过双向分析文本，能够深入理解词语在句子中的完整上下文含义**

核心在于**双向**分析文本`->`上下文理解，类似于“完形填空”

重大意义就是正式确立了**预训练-微调**的两阶段思想

BERT 有两种规模的模型。分别是

**base 版本**（12层 Encoder Layer，768 的隐藏层维度，总参数量 110M）

**large 版本**（24层 Encoder Layer，1024 的隐藏层维度，总参数量 340M）

*******



**核心内容**

1. **架构组成**：BERT 本质上是由 **Transformer Encoder** 堆叠而成的。它由 **Embedding层**、**N层 Encoder Layer**（Base版12层/Large版24层）和用于适配下游任务的 **Prediction Heads（分类头）** 组成。
2. **核心机制与改进**：
   - **注意力机制**：沿用了自注意力（Self-Attention），但改进版引入了**可训练的相对位置编码**，比 Transformer 的绝对位置编码更能拟合位置信息（但受限于 512 token 的最大长度）。
   - **激活函数**：普遍采用了 **GELU**（高斯误差线性单元），引入随机正则化思想。
3. **文本处理**：采用 **WordPiece** 子词切分算法；对于中文，通常以**单个汉字**作为基本单位（token）。
4. **任务适配**：BERT 专注于 **NLU（自然语言理解）** 任务。通过将顶层隐藏状态输入 Prediction Heads（线性层+激活函数），将高维特征映射为具体类别的概率输出。



**为什么说 BERT = Transformer Encoder？**

当我们说 BERT 等于 Transformer 架构时，指的是它完整继承了 Encoder 端的**并行计算特征**：

- **完全相同的注意力机制**：BERT 内部的 `Self-Attention` 模块，在代码实现上和 Transformer 的 Encoder 模块一模一样。
- **“全连接”特性**：Transformer Encoder 的设计初衷就是为了让每一个输入词都能看到其他所有词（为了更好地做特征提取）。BERT 只是把这个特性发挥到了极致，用来做**预训练**

| 特性            | Transformer (标准)   | BERT                 |
| :-------------- | :------------------- | :------------------- |
| **主要组件**    | Encoder + Decoder    | **仅 Encoder**       |
| **掩码 (Mask)** | Decoder 部分有 Mask  | **完全没有 Mask**    |
| **任务目标**    | 翻译、总结 (Seq2Seq) | 理解、分类、特征提取 |

******



**核心细节了解**

1. 双向注意力机制（Bidirectional Attention）
   - **本质区别**：BERT 是 **Encoder-only**。它的每个 Token 都能看到序列中“左边”和“右边”的所有信息（计算 Self-Attention 时没有 Mask）。
   - **开发意义**：这决定了它极其适合做**理解**（NLU）和**特征提取**（Embedding），但完全不适合做生成。在做 **RAG（检索增强生成）** 的检索环节时，必须选这种双向架构的模型。
2. 位置编码：从绝对到相对（Positional Embedding）
   - **细节**：原始 BERT 是**绝对位置编码**（给 1-512 每个位置一个索引）。
   - **局限性**：这导致它有**外推性差**的死穴。如果你训练时用 512 长度，推理时遇到 513 长度，模型会直接“报废”。
   - **进阶**：现在主流 LLM 用的 **RoPE（旋转位置编码）** 或 **ALiBi** 都是为了解决 BERT 留下的这个长度限制难题。理解了 BERT 的痛点，你才能理解为什么 Llama 处理长文本要改位置编码。

3. WordPiece 分词逻辑（Tokenizer）
   - **逻辑**：它把词拆成子词（Subword），比如 `embeddings` 拆成 `em` + `##bed` + `##dings`。
   - **实战关联**：
     - **##号的含义**：表示该子词不是单词开头。
     - **未登录词处理**：为什么模型不会出现 OOV（词汇表外）错误？因为最后会拆成单字符。
     - **中文分词**：BERT 默认对中文是**按字分词**。如果你做垂直领域（如医疗、法律）的增量预训练，是否需要**扩充词表**或**更改分词策略**是性能优化的关键。

4. 特殊标记（Special Tokens）
   - **[CLS]**：放在开头，它的输出向量被认为代表了整个句子的语义（用于分类）。
   - **[SEP]**：句子的分隔符。
   - **[MASK]**：预训练的核心。
   - **避坑指南**：在做大模型微调时，如果你漏掉了这些 Token 或在处理单句时错误地使用了 [SEP]，会导致模型性能大幅下降。

5. GELU 激活函数
   - **对比**：相比 ReLU（在负数区直接“死掉”，梯度为0），GELU 在 0 点附近是平滑的且允许少量负值通过。
   - **意义**：这种**平滑性**是大模型能训练得更深、更稳定的基础。现在几乎所有主流 LLM（GPT-4, Llama 3 等）都沿用了这种放弃 ReLU 转向平滑激活函数的思路。

******



**创新点**

相较于基本沿承 Transformer 的模型架构

BERT 更大的创新点在于其提出的两个新的预训练任务上——MLM 和 NSP

**MLM (Masked Language Model)：掩码语言模型**

**NSP (Next Sentence Prediction)：下一句预测**

其中

**LM 任务**：像在写小说（生成式）。

**MLM 任务**：像在做阅读理解（理解式）



BERT 放弃了传统的 **LM（从左到右预测下文）**，因为它觉得那样会漏掉右边的信息。

1. MLM (掩码语言模型) —— 解决“双向理解”
   - **核心：** 完形填空。把句子里的词遮住（Mask），让模型结合上下文猜这个词。
   - **双向性：** 因为模型能看到左边，也能看到右边，所以它对词语的理解是全方位的（这比单向的 GPT 路线在理解任务上更强）。
   - **15% 的策略（面试高频）：** 为了解决“预训练时有 Mask，微调时没有 Mask”的不一致问题，BERT 玩了个花活：
     - **80%** 真的换成 `[MASK]`。
     - **10%** 随机换成别的词（迫使模型怀疑每一个词，时刻保持警惕）。
     - **10%** 保持原样（告诉模型：大部分词是对的，你要学会提取特征）。

2. NSP (下一句预测) —— 解决“句间逻辑”
   - **核心：** 逻辑判断。给模型两个句子，问它：B 是不是接在 A 后面的？
   - **意义：** 像问答、推理这种任务，光懂词的意思不行，得懂句子之间的因果和转折关系。

### RoBERTa

RoBERTa是基于 BERT 的优化模型

RoBERTa 的模型架构与 BERT 完全一致，也就是使用了 BERT-large



**优化**

1. 任务进化：做减法，去掉了 NSP

   RoBERTa 最具争议也最成功的改动就是**废除了“下一句预测”（NSP）**。

   - **实验结论：** 学者发现 NSP 任务对模型提升很小，甚至有负面影响。
   - **最佳方案：** 采用**单文档 MLM**。即输入不再是强行拼接的“句子对”，而是从同一个文档中连续采样，直到填满 512 个 Token。
   - **意义：** 这样模型能接触到更长、更连贯的上下文，而不是被琐碎的句子对打断。

2. 策略进化：静态变动态 (Dynamic Masking)

   - **BERT (静态)：** 数据预处理时就定死了哪些词被遮住，模型在 40 个 Epoch 里反复看同样的“填空题”，容易死记硬背。
   - **RoBERTa (动态)：** 在**训练阶段**实时遮蔽。这意味着每一轮（Epoch）模型看到的遮盖位置都不同。
   - **意义：** 虽然性能提升微弱，但让训练过程更鲁棒、更高效，成了后续模型的标配。

3. 资源进化：大力出奇迹 (Scale Up)

   RoBERTa 验证了“更大数据、更大 Batch、更多步数”的威力：

   - **数据量：** 从 16GB 暴增到 **160GB**（包含新闻、网页、故事等多种语料）。
   - **Batch Size：** 从 256 提升到 **8000 (8K)**。大 Batch 既能加速训练，又能让模型学得更稳、最终效果更好。
   - **序列长度：** 放弃了 BERT 先练短句再练长句的折中办法，全程采用 **512 全长度**训练。
   - **代价：** 这种“炼丹”需要 1024 块 V100 显卡连跑一天，是真正的算力竞赛。

4. 编码进化：更广的词表 (BPE)
   - **算法切换：** 从 WordPiece 切换到 **Byte-level BPE**（字节对编码）。
   - **词表扩大：** 词典规模从 30K 扩大到 **50K**。
   - **意义：** 词表越大，模型对生僻词和字符的表征能力越强。虽然增加了 Embedding 层的参数量，但换来了更好的编码质量。

### ALBERT

ALBERT 的全称是 **A** **L**ite **BERT**（精简版 BERT）。它的核心目标是：在不损失（甚至提升）性能的前提下，大幅减少 BERT 的参数量，让模型变得更轻、更快



**优化**

1. 结构减负：Embedding 因式分解

   ALBERT 敏锐地察觉到了 BERT 在显存分配上的一个“浪费”：

   - **痛点：** 在 BERT 中，词向量维度（E）必须等于隐藏层维度（H）。如果你想让模型更聪明（增加 H），词表参数就会爆炸式增长。
   - **解法：** 把 $V \times H$ 的大矩阵拆成 $V \times E$ 和 $E \times H$ 两个小矩阵。
   - **直观理解：** 先用较小的维度（如 128）把词的意思表达出来，再通过一个线性变换“放大”到隐藏层维度（如 1024）。
   - **价值：** 成功让 Embedding 参数量从 **30M 降到了几 M**。

2. 逻辑提炼：跨层参数共享 (Cross-layer Parameter Sharing)

   这是 ALBERT 最惊世骇俗的改动，也是它“Lite”的真正原因。

   - **做法：** 别管是 12 层还是 24 层 Encoder，它们全都**共用同一套权重参数**。
   - **本质：** BERT 是 12 个人接力跑；ALBERT 是一个人绕着操场跑 12 圈。
   - **优缺点对决：**
     - **优点：** 参数量暴减（只有原来的约 1/10）。这让模型可以变得“极宽”（隐藏层维度拉到 2048 甚至更高），而显存占用依然很低。
     - **致命伤（导师敲黑板）：** **速度没变快！** 虽然参数少了，但计算次数（那 24 圈）没少。所以它省显存，但不省时间。

3. 任务升级：从 NSP 到 SOP (句子顺序预测)

   ALBERT 认为 RoBERTa 直接删掉 NSP 太可惜了，问题不在于任务本身，而在于题太简单。

   - **NSP 的漏洞：** 模型只要发现两个句子聊的主题不一样（比如一个聊足球，一个聊吃饭），就能秒杀。这叫“主题预测”，不叫“逻辑理解”。
   - **SOP 的改进：** 正样本是“A+B”，负样本是**把顺序反过来的“B+A”**。
   - **价值：** 模型被迫去理解“因为...所以...”这种细微的逻辑先后顺序。实验证明，这种“变态级”的逻辑训练让模型在下游任务中变得更聪明。

## GPT

一种自然语言处理（NLP）模型，它通过**单向（从左到右）**分析文本，核心能力是根据上文预测下一个词，从而具备极强的文本生成和逻辑推理能力。

- **核心在于：** 单向因果分析 -> 文本生成，类似于“接龙游戏”。
- **重大意义：** 证明了 **Scale Law（规模法则）** 的力量，确立了从“预训练-微调”向“预训练-提示工程（Prompt）”转化的范式。



**核心内容**

- **架构组成：** GPT 本质上是由 **Transformer Decoder** 堆叠而成的。它由 Embedding 层、N 层 Decoder Layer（仅保留自注意力层）和线性输出层组成。
- **核心机制与改进：**
  - **掩码自注意力（Masked Self-Attention）：** 这是与 BERT 的本质区别。每个 Token 只能看到它左边的信息，右边的信息被掩码（Mask）遮蔽，确保生成时的因果律。
  - **位置编码：** GPT-1 沿用 Sinusoidal 绝对位置编码；GPT-2/3 引入了可训练的位置编码。
  - **归一化顺序（Pre-Norm）：** GPT-2 开始将 LayerNorm 提到注意力层之前。这就像在长跑前先做拉伸，保证了模型在参数量达到千亿级时，梯度依然稳定。
  - **激活函数：** 同样采用 **GELU**，确保深度网络的平滑性。

------

**为什么说 GPT = Transformer Decoder (Simplified)？**

GPT 舍弃了 Transformer 原始设计中复杂的 Encoder-Decoder 交互，只留下了 Decoder：

- **舍弃 Cross-Attention：** 因为没有 Encoder 的输入，所以 GPT 的每一层只专注于处理自己的输入序列。
- **单向并行：** 训练时可以利用掩码矩阵实现并行计算，但在推理（生成）时，必须一个词一个词地往后蹦（自回归生成）。

| **特性**       | **BERT**             | **GPT**                             |
| -------------- | -------------------- | ----------------------------------- |
| **主要组件**   | 仅 Encoder           | 仅 Decoder                          |
| **注意力方向** | 双向（看到全句）     | **单向（仅看左侧）**                |
| **掩码机制**   | 随机遮盖（15% Mask） | **因果遮盖（未来 Token 全部屏蔽）** |
| **擅长领域**   | 分类、标注、语义理解 | **创作、对话、代码、逻辑推理**      |

------

**核心细节了解**

1. 掩码自注意力机制（Masked Self-Attention）
   - **本质区别：** 它是 **Causal（因果）** 的。在计算第 $i$ 个词的注意力时，会将 $i+1$ 到 $n$ 的权重全部设为负无穷。
   - **开发意义：** 这决定了它能“无中生有”生成文本。在做**聊天机器人、文本续写、创意写作**时，GPT 路线是唯一选择。

2. 从 Post-Norm 到 Pre-Norm 的演进
   - **细节：** GPT-1 像原始 Transformer 一样把 Norm 放在后面；GPT-2 开始挪到前面。
   - **实战关联：** 随着层数（48层到96层）的叠加，Pre-Norm 极大地缓解了梯度消失问题。这是大模型能“大”起来的技术前提。

3. BPE (Byte Pair Encoding) 分词逻辑
   - **逻辑：** 相比 BERT 的 WordPiece，GPT-2/3 使用的 BPE 更加精细，通常基于字节级别。
   - **优势：** 进一步减少了 `[UNK]` (未知词) 的出现。无论你输入什么古怪的符号，BPE 都能将其拆解成模型认得的字节组合。

4. 从 Zero-shot 到 Few-shot (In-context Learning)
   - **Zero-shot：** 直接下指令（例如：“翻译这段话：...”）。
   - **Few-shot (ICL)：** 在指令后给 3-5 个例子。
   - **底层逻辑：** GPT-3 证明了，当参数量够大时，模型不需要改动权重，只需通过**上下文提示**就能激活内部潜能。这催生了 **Prompt Engineering（提示工程）** 这一新职业。

------

**创新点：预训练任务 —— CLM (Causal Language Model)**

GPT 放弃了 BERT 的 MLM（填空）和 NSP（下句预测），只玩一种最极致的任务：

- **CLM (因果语言模型)** —— **解决“逻辑延续”**
  - **核心：** 预测下一个词。给模型 $N$ 个词，让它猜第 $N+1$ 个。
  - **优势：**
    1. **数据无尽：** 只要有文字，就是天然的标签，不需要任何人工标注。
    2. **逻辑一致：** 训练时在续写，用的时候也在续写，**预训练与下游任务完全对齐**。
  - **涌现能力：** 当模型规模（参数、数据、算力）突破临界点时，GPT 突然学会了本没教过它的复杂逻辑推理和零样本迁移能力。

------

**避坑指南**

1. **别用 GPT 做 Embedding：** 虽然它也能出向量，但因为是单向的，它的语义表征通常不如 BERT 这种双向模型深厚。
2. **幻觉问题（Hallucination）：** 由于 CLM 只是在猜下一个词的“概率”，它有时会极其自信地胡编乱造。这是生成式模型（Decoder-only）天生的弱点。
3. **计算代价：** 生成任务是逐个 Token 产生的，KV Cache 等优化技术是你在开发高并发 GPT 应用时必须掌握的。

### LLaMA

一种由 Meta 开发的开源 **Decoder-only** 模型。

- **核心在于：** 在 GPT 的基础上，通过“微调”架构细节，配合超大规模（T 级别）的语料，实现了模型能力与参数规模的非线性跨越。
- **重大意义：** 它的开源彻底改变了大模型生态，让开发者可以在自己的服务器（甚至笔记本）上跑出接近 GPT-4 的效果。



**架构组成：** 延续了 **Transformer Decoder** 的堆叠结构。由 Embedding 层、由 masked self-attention 和 MLP 组成的 Decoder Blocks，以及最后的线性输出层构成。

**核心机制改进（LLaMA 的“三板斧”）：**

- **位置编码改进：** 放弃了 GPT 的绝对位置编码，采用 **RoPE（旋转位置编码）**，提供了更好的长度外推性。
- **归一化改进：** 采用 **RMSNorm** 并在层前进行归一化（Pre-Normalization），提升了训练的极其稳定性。
- **激活函数改进：** 将传统的 ReLU 或 GELU 换成了 **SwiGLU**，显著提升了模型的非线性表达能力。

### GLM

一种旨在**统一 NLU（理解）和 NLG（生成）**的通用语言模型架构。

- **核心在于：** 自回归空白填充（Autoregressive Blank Infilling）。它把句子抠掉一段，让模型既要通过上下文“理解”该填什么，又要通过自回归“生成”出来。
- **重大意义：** 它是国产大模型（清华/智谱系列）的基石。虽然后期回归了 CLM 路线，但它在模型小型化、量化普及和中文原生支持上对国内开发者贡献巨大



**核心内容**

- **架构组成：** 基础仍是 **Decoder-Only**。
- **与 GPT 的三点细微差异（技术修正）：**
  1. **归一化顺序（Post-Norm）：** 早期 GLM 坚持在残差之后做归一化。虽然 Pre-Norm 更易训练，但 Post-Norm 正则化效果更强，模型更鲁棒。
  2. **预测层简化：** 最终 Token 预测使用**单个线性层**而非 MLP，减少了输出端的参数冗余。
  3. **激活函数：** 采用 **GELU**。相比 ReLU，它在 0 点附近的连续非线性映射让梯度传播更平滑。

**创新点**

预训练任务 —— 融合 MLM 与 CLM

这是 GLM 最具辨识度的标签，即“自回归空白填充”：

- **自编码思想（来自 MLM）：** 随机遮蔽一连串 Token（Span Masking），迫使模型双向理解上下文。
- **自回归思想（来自 CLM）：** 在预测被遮蔽的部分时，按顺序一个词一个词往外蹦（Reconstruct）。
- **优势：** 既能像 BERT 一样看全局（解决阅读理解），又能像 GPT 一样续写（解决生成）。
