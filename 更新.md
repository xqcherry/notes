# 较上次的改动

1. 目前所有逻辑都接入鉴权，测试接口必须携带Bearer Token
2. 权限设计结构为：区分了普通用户和root用户，允许普通用户创建团队，进行团队crud操作，允许在团队内进行任务crud操作
3. 任务创建操作添加了teamid字段
4. 完整操作链为：团队创建->任务创建与派发
5. 修改派发ALL的字段范围为团队内成员
6. 目前root功能无特别设计，只作为区分，原先设计的管理员功能即现在用户创建团队后自动成为队长（管理员）的功能
7. 新增了设置 root 管理员接口和一系列团队接口，因为目前root功能无特别设计，只作为区分，所以此接口仅作为添加的功能
8. root用户不参与团队功能，禁止成为团队成员
9. 创建任务的items字段已分为"problemItems""contestItems""solutionItems"
10. 添加了周常任务逻辑，详细用法在另一个md
11. 周常任务的设计逻辑是先创建任务（多携带两个字段判定为周常任务），创建的任务作为模板，派发任务的逻辑是每周自动copy出新的任务派发下去，所以本质上一开始创建的是模板任务，后面的都是模板任务自动产生的实例，派发的都是实例的taskId，而不是一开始创建的模板任务的taskId，对实例/模板任务的查询接口都同普通的查询任务
